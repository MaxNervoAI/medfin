## Critical Issues

### 1. Not Mobile-First (BLOCKER)
- Only 1 breakpoint at 900px
- No Tailwind responsive prefixes (`sm:`, `md:`, `lg:`)
- Fixed pixel widths everywhere
- No bottom tab navigation
- Tables don't adapt to mobile

### 2. Inline Styles Crisis (BLOCKER)
- `AppShell`, `DashboardClient`, `PresupuestoClient`, `InstitucionesClient` use inline `style={{}}`
- `NuevaPrestacionForm`, `PrestacionDetalle` use Tailwind classes
- Two conflicting systems = unmaintainable
- Cannot do responsive design with inline styles

### 3. Missing Production Elements
- Zero tests
- No error boundaries
- No loading skeletons
- No offline capability
- No analytics
- No rate limiting

## Roadmap to Production

### Phase 1: Foundation (2-3 weeks)
- Install + configure `next-pwa`
- Create `tax_settings` table, remove hardcoded `14.5`
- Add `inputMode="numeric"`
- Configure Vercel cron + Resend domain

### Phase 2: Mobile-First Refactor (3-4 weeks)
- Replace ALL inline styles with Tailwind responsive classes
- Implement bottom tab bar for mobile
- Add breakpoints (`sm:`, `md:`, `lg:`) to all components
- Convert dashboard table to cards on mobile
- Optimize touch targets (min 44px)

### Phase 3: Hardening (2 weeks)
- Error boundaries + global error handling
- Loading skeletons
- E2E tests (Playwright)
- API rate limiting
- Edit prestacion + boleta number + month filter

### Phase 4: Polish (1 week)
- SEO, analytics, performance, accessibility

**Total: 8-10 weeks to production-ready mobile-first PWA.**
